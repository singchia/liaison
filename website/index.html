核心优势

从安全、体验到运维效率，面向真实互联网环境打造。

安全可靠

TLS 加密传输，内网不暴露；支持 JWT 鉴权与权限边界。

  • 连接建立可控，可随时开启关闭
  • 面向公网访问场景的默认安全策略

简单易用

控制台完成全流程操作；一键生成安装命令，秒级上线连接器。

  • 统一入口管理连接器、设备、应用与代理
  • 快速开始与脚本分发开箱即用

自动发现

设备上报 + 远程扫描闭环，自动发现内网应用与端口。

  • 减少手动录入与错误配置
  • 适配家庭、企业与边缘场景

跨平台分发

连接器支持 Linux/macOS/Windows，覆盖 x86_64 与 ARM64，适用于服务器、工控与边缘设备。

可视化监控

设备在线状态、延迟、流量统计可视化呈现，为运维与容量评估提供数据依据。

架构一目了然

中心化控制平面统一管理，边缘连接器负责最后一公里的通达。

Manager

控制台 + API：连接器管理、代理配置、任务下发、数据可视化。

Frontier

连接器网关：统一承载连接与通信，简化公网入口与运维。

Edge

边缘连接器：设备信息上报、端口扫描、流量采集与应用转发。

典型访问路径
互联网客户端 → 代理端口（Entry）→ 连接器（Edge）→ 内网应用 (ip:port)

常见问题

面向互联网部署时最常见的疑问。

Liaison 和传统“内网穿透”有什么区别？
Liaison 以“管理中心 + 连接器 + 代理入口”的体系化方式提供可视化控制、任务下发、自动发现与流量统计，适合管理多设备、多应用的长期场景。
是否会暴露内网地址与拓扑？
默认策略强调不暴露内网，外部访问通过代理端口完成转发，并提供 TLS 加密保障链路安全。
适合个人使用还是企业使用？
两者都适合：个人可用于 NAS/远程办公；企业可用于多机房通达、边缘设备管理与运维监控。
